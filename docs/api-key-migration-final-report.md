## ⚠️ 技术债务和风险

### 1. 类型兼容性问题 (P2)

**问题**:
- Better Auth 类型定义不完整
- 需要使用 `as any` 绕过类型检查

**临时方案**:
```typescript
const result = await (auth.api as any).createApiKey({ ... });
```

**长期方案**:
- 等待 Better Auth 官方修复
- 或提交 PR 完善类型

### 2. 数据迁移风险 (P1)

**问题**:
- Hash 算法不同，无法直接迁移
- 必须为每个用户重新生成 Key

**应对措施**:
- ✅ 提前 2 周通知用户
- ✅ 提供过渡期（2 周）
- ✅ 自动发送新 Key
- ✅ 一键重新生成流程

### 3. API Key Service 未删除 (P1)

**问题**:
- `api-key.service.ts` 文件仍然存在
- 可能导致混淆

**解决方案**:
- Week 4 确认功能正常后删除
- 或重命名为 `.deprecated.ts`
