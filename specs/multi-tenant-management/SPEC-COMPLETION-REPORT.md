### 3. ✅ 改进 .opencode/commands/spec.md

**改进内容**：

#### 3.1 增强 `sync` 操作

```markdown
# 新增内容

3. **如果目录存在**，更新：
   - 读取现有 design.md
   - 扫描代码变更
   - 更新 implementation.md 的已完成项
   - 记录新的决策到 decisions.md
   - **检查并更新模板文档**： ← 新增
     - 如果 testing.md 还是模板 → 填充实际测试计划
     - 如果 workflow.md 还是模板 → 填充实际开发流程
     - 如果 prompts.md 还是模板 → 填充常用提示词
     - 如果 future-work.md 还是模板 → 填充后续工作
```

#### 3.2 添加自动触发条件

定义了 **5 个触发条件**：

1. 功能已开始开发
2. 存在测试文件
3. 存在已实现的源代码
4. Phase 1 完成
5. Phase 2 完成

#### 3.3 添加 `fill` 操作

```bash
# 新命令
/spec fill <feature-name> [document-name]
```

**用途**：手动填充模板文档

#### 3.4 改进 `list` 操作

新增**模板状态列**：

- ✅ 已填充
- ⚠️ 部分模板
- ❌ 未填充

#### 3.5 添加填充详细指南

为每个模板文档添加了详细的填充指南：

- 何时填充
