# Workflow 翻译

## 概述

使用 lingo.dev 将 workflow 通知自动翻译为参会人的浏览器语言。

## 使用方式

### 第 1 步：启用自动翻译

编辑 workflow step（`EMAIL_ATTENDEE`、`SMS_ATTENDEE` 或 `WHATSAPP_ATTENDEE`）时，开启 “Auto-translate for attendees”。

### 第 2 步：生成翻译

保存 workflow 后，系统会自动生成 19 种语言的翻译。

### 第 3 步：参会人接收翻译后的通知

创建预订后，参会人会收到其浏览器语言的通知（若该语言受支持）。

## 配置选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| Auto-translate for attendees | 启用/禁用翻译 | Off |

## 常见问题

### 支持哪些语言？

英语、西班牙语、德语、葡萄牙语、法语、意大利语、阿拉伯语、俄语、中文（简体与繁体）、日语、韩语、荷兰语、瑞典语、丹麦语、冰岛语、立陶宛语、挪威语。

### 如果参会人语言不受支持会怎样？

会收到原始语言（编写时语言）的通知。

### 模板变量会被翻译吗？

不会。`{EVENT_NAME}` 这类变量会被保留，并在翻译后再替换。
