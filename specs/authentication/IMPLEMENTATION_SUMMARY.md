## 五、技术决策

### 决策 1：使用内存缓存作为默认方案

**理由：**
- 开发环境无需额外依赖
- 性能足够（查询时间 < 0.1ms）
- 配置简单

**风险：**
- 单实例限制
- 重启后缓存丢失

**缓解：**
- 生产环境推荐启用 Redis
- 提供降级方案

### 决策 2：Redis 不可用时自动降级

**理由：**
- 提高系统可用性
- 避免单点故障
- 平滑迁移

**实现：**
- 检测 Redis 连接状态
- 自动切换到内存缓存
- 记录日志告警

### 决策 3：速率限制使用内存存储

**理由：**
- @nestjs/throttler 默认方案
- 性能足够（单实例）
- 配置简单

**风险：**
- 分布式环境不准确

**缓解：**
- 未来可升级到 Redis 存储
- 当前优先保护单实例
