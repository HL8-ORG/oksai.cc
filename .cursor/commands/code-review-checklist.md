# 代码审查清单

## 概述
用于进行全面代码审查的综合清单，确保代码质量、安全性和可维护性。

## 审查类别

### 项目规范（AGENTS.md）
- [ ] 代码注释、文档、错误消息使用中文
- [ ] Git 提交信息使用英文
- [ ] 公共 API / Controller / Service 有完整 TSDoc 注释
- [ ] 代码变更时同步更新了相关注释
- [ ] 遵循 Nx monorepo 工作流（使用 `pnpm nx` 而非直接工具）

### Spec 优先开发
- [ ] 新功能开发前已创建 spec 文档（`specs/{feature}/design.md`）
- [ ] 实现进度已记录在 `implementation.md`
- [ ] 架构决策已记录在 `decisions.md`（如有重要决策）
- [ ] 代码实现与 spec 设计保持一致

### 功能性
- [ ] 代码实现了预期功能
- [ ] 边界情况已处理
- [ ] 错误处理得当
- [ ] 无明显 bug 或逻辑错误

### 代码质量
- [ ] 代码可读性好，结构清晰
- [ ] 函数短小且职责单一
- [ ] 变量命名具有描述性
- [ ] 无代码重复
- [ ] 遵循项目规范

### 安全性
- [ ] 无明显安全漏洞
- [ ] 存在输入验证
- [ ] 敏感数据处理得当
- [ ] 无硬编码的密钥
