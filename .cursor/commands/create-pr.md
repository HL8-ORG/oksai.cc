# 创建 PR

## 概述
创建一个结构良好的拉取请求，包含适当的描述、标签和审查者。

## 步骤
1. **准备分支**
   - 确保所有更改已提交
   - 将分支推送到远程仓库
   - 验证分支与主分支保持同步

2. **撰写 PR 描述**
   - 清晰地总结更改内容
   - 包含上下文和动机说明
   - 列出任何破坏性更改
   - 如有 UI 更改，添加截图

3. **设置 PR**
   - 使用遵循规范 PR 格式的描述性标题创建 PR，例如：feat(scope): 添加炫酷新功能
      - 作用域列表为项目当前的作用域：dashboard、worker、api-service、js、react 等...
   - 添加适当的标签
   - 指定审查者
   - 在 Linear 上关联相关议题
